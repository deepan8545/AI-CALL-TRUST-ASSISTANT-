# Claude Code Rules — AI Call Trust Assistant

## Rule 1: Plan mode first
Use Plan Mode for anything complex. Analyze the codebase and propose the smallest safe implementation plan before writing code.

## Rule 2: One step at a time
Do not implement an entire system in one prompt. Break work into focused, testable steps.

## Rule 3: Preview before accepting
Always inspect diffs before applying changes.

## Rule 4: Let Claude fix its own bugs
When a test fails, explain the root cause first, then propose the smallest fix. Do not change unrelated files.

## Rule 5: Run simplify before review
Remove over-engineering and keep only what is needed for the MVP.

## Rule 6: End every session with a retro
Save learnings to docs/claude-code-session-notes.md.

## Architecture Rules
- The LLM explains decisions; it does NOT make the primary fraud decision.
- Real-time scoring uses: Rules → Redis → ML → Neo4j graph features.
- All sensitive identifiers must be hashed.
- Do not store raw audio without explicit consent and legal approval.
- Use MockLLMAdapter for all tests — never call real LLM APIs in tests.
- Every service has its own Dockerfile.
- PostgreSQL for transactional data, Neo4j for relationship intelligence.
